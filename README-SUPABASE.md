# 🚀 Despliegue con Supabase

## Pasos para configurar Supabase

### 1. Crear proyecto en Supabase

1. Ve a [supabase.com](https://supabase.com)
2. Click en **"Start your project"**
3. Elige un nombre para tu proyecto
4. Elige una región (preferiblemente cerca de tu ubicación)
5. Configura la contraseña de master
6. Click en **"Create new project"**

### 2. Configurar base de datos

Una vez creado el proyecto:

1. Ve al menú lateral → **Database**
2. Verás la URL de conexión (usa la que muestra Supabase)
3. **Importante**: Supabase crea automáticamente:
   - Usuario: `postgres`
   - Password: (tu password de master)
   - Database: `postgres`

### 3. Obtener credenciales

En el Dashboard de Supabase:

1. **Settings** (engranaje)
2. **API**
3. Copia las credenciales:
   - **Project URL** → `VITE_SUPABASE_URL`
   - **API Key (anon)** → para frontend (lectura)
   - **API Key (service_role)** → para backend (escribe)

```env

```

### 5. Configurar permisos en Supabase

#### Rol: anon (frontend)
- Puede: leer, listar, filtros básicos
- No puede: crear, borrar, filtrar por usuario

#### Rol: authenticated (usuarios logueados)
- Puede: crear, editar sus propios registros
- No puede: ver registros de otros usuarios

#### Rol: service_role (backend interno)
- Puede: todo
- Solo para el backend del mismo proyecto

### 6. Crear tabla de inventario

En el **SQL Editor** de Supabase:

```sql
-- Tabla de productos
create table products (
  id bigint generated by default as identity primary key,
  name varchar not null,
  description text,
  sku varchar unique,
  quantity integer default 0,
  min_quantity integer default 5,
  price numeric(10,2) default 0,
  cost numeric(10,2) default 0,
  status varchar default 'active' check (status in ('active','inactive','discontinued')),
  created_at timestamp with time zone default timezone('utc'::text, now()),
  updated_at timestamp with time zone default timezone('utc'::text, now())
);

-- Tabla de categorías
create table categories (
  id bigint generated by default as identity primary key,
  name varchar not null unique,
  description text,
  parent_id bigint references categories(id),
  icon varchar,
  sort_order integer default 0
);

-- Tabla de movimientos
create table movements (
  id bigint generated by default as identity primary key,
  product_id bigint references products(id),
  quantity integer not null,
  movement_type varchar not null check (movement_type in ('in','out','adjustment','transfer')),
  reference varchar,
  reference_type varchar,
  notes text,
  created_by varchar,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- Tabla de vencimientos
create table expirations (
  id bigint generated by default as identity primary key,
  product_id bigint references products(id),
  lot_number varchar,
  quantity integer not null,
  expiration_date date not null,
  batch_code varchar,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- Trigger para actualizar updated_at
create or replace function update_updated_at_column()
returns trigger as $$
begin
  new.updated_at := timezone('utc'::text, now());
  return new;
end;
$$ language plpgsql;

-- Aplicar trigger a las tablas
alter table products add column updated_at timestamp with time zone default timezone('utc'::text', now());
alter table categories add column updated_at timestamp with time zone default timezone('utc'::text', now());
alter table movements add column updated_at timestamp with time zone default timezone('utc'::text', now());
alter table expirations add column updated_at timestamp with time zone default timezone('utc'::text', now());

create or replace function update_timestamps() returns trigger as $$
begin
  update products set updated_at = now() where id = new.id;
  update categories set updated_at = now() where id = new.id;
  update movements set updated_at = now() where id = new.id;
  update expirations set updated_at = now() where id = new.id;
  return new;
end;
$$ language plpgsql;

-- Reemplazar el trigger anterior
drop trigger if is_updated_at on products;
create trigger is_updated_at on products after insert or update execute procedure update_updated_at_column();
drop trigger if is_updated_at on categories;
create trigger is_updated_at on categories after insert or update execute procedure update_updated_at_column();
drop trigger if is_updated_at on movements;
create trigger is_updated_at on movements after insert or update execute procedure update_updated_at_column();
drop trigger if is_updated_at on expirations;
create trigger is_updated_at on expirations after insert or update execute procedure update_updated_at_column();
```

### 7. Configurar Row Level Security (RLS)

```sql
-- Habilitar RLS
alter table products enable row level security;
alter table categories enable row level security;
alter table movements enable row level security;
alter table expirations enable row level security;

-- Políticas para anon (frontend público)
create policy "anon_read_products" on products for select using (true);
create policy "anon_read_categories" on categories for select using (true);
create policy "anon_read_movements" on movements for select using (true);
create policy "anon_read_expirations" on expirations for select using (true);

-- Políticas para authenticated (usuarios logueados)
create policy "auth_create_products" on products for insert with check (auth.role() = 'authenticated');
create policy "auth_own_products" on products for select using (auth.uid() = creator_id);
-- etc...
```
